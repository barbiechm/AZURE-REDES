Guía Completa de Azure VNET

Configuración profesional de redes virtuales en Azure

1. Interfaces de Red (NIC)

+

¿Qué es una NIC en Azure?

Una interfaz de red (NIC) es el componente fundamental que permite a tu máquina virtual (VM) conectarse a una red virtual (VNET) y comunicarse con otros recursos.

Cuando creas una VM en Azure, se le asigna automáticamente una NIC con las siguientes características:

  • Dirección IP privada (asignada por defecto desde el rango de la subred)
  • IP pública opcional (para comunicación con Internet)
  • Asociación con grupos de seguridad de red (NSG) para controlar el tráfico
  • Conexión a una subred específica dentro de tu VNET
ℹ️

Nota importante: Una VM puede tener múltiples NICs, cada una conectada a diferentes subredes, lo que permite configuraciones de red avanzadas.

Configuración de una NIC

Puedes configurar una NIC durante la creación de la VM o posteriormente:

  • Durante la creación: Azure configura automáticamente la NIC básica
  • Posteriormente: Puedes añadir NICs adicionales o modificar las existentes

2. Configuración IP (IPv4/IPv6)

+

IPv4 en Azure

IPv4 es el protocolo de Internet más utilizado en Azure:

  • Asignación automática de IP privada desde el rango de la subred
  • Puede ser dinámica (cambia al reiniciar) o estática (fija)
  • Soporte para IP públicas asignadas a recursos

IPv6 en Azure

Azure también soporta IPv6, pero requiere configuración adicional:

  • Debe habilitarse explícitamente en la VNET
  • Configuración especial necesaria para la subred
  • Compatibilidad con la mayoría de servicios de Azure
⚠️

Atención: No todos los servicios de Azure soportan IPv6 completamente. Verifica la documentación antes de implementar.

3. Redes Virtuales (VNET)

+

¿Qué es una VNET?

Una red virtual (VNET) es el bloque fundamental para tu red privada en Azure:

  • Equivalente a una VPC en AWS
  • Aislamiento lógico completo de otros recursos
  • Puede dividirse en múltiples subredes
  • Conecta VMs, bases de datos y otros servicios

Características clave

  • Espacio de direcciones: Define el rango IP de toda la red
  • Subredes: Divisiones lógicas dentro de la VNET
  • Grupos de seguridad: Controlan el tráfico entrante/saliente
  • Conectividad: Puede conectarse a otras VNETs o a redes locales

4. Configuración DNS

+

Opciones DNS en Azure

Azure ofrece varias opciones para la resolución de nombres:

  • DNS de Azure: Servicio interno proporcionado por defecto
  • DNS personalizado: Usa tus propios servidores (Google DNS, Cloudflare, etc.)
  • Azure DNS: Servicio de alojamiento de dominios con alta disponibilidad

Zonas DNS

Puedes crear zonas DNS públicas o privadas:

  • Zonas públicas: Para dominios accesibles desde Internet
  • Zonas privadas: Para resolución de nombres dentro de tu VNET

5. Balanceadores de Carga

+

Load Balancer (Nivel 4)

Balanceador básico para tráfico TCP/UDP:

  • Distribuye tráfico basado en IP y puerto
  • Ideal para tráfico no HTTP
  • Puede ser interno o público

Application Gateway (Nivel 7)

Balanceador avanzado para aplicaciones web:

  • Enrutamiento basado en URL
  • Terminación SSL (SSL offloading)
  • Firewall de aplicaciones web (WAF)
  • Affinity de sesión
💡

Recomendación: Usa Application Gateway para aplicaciones web y Load Balancer para otro tipo de tráfico.

Flujo Básico para Servidor Web

+
1

Crea una VNET con subred

Define el espacio de direcciones y las subredes necesarias para tu aplicación.

2

Despliega una VM en la subred

Selecciona la imagen adecuada para tu servidor web (Windows IIS, Linux Apache/Nginx, etc.).

3

Asigna NIC con IP privada/pública

Configura la interfaz de red con una IP privada y, si es necesario, una IP pública.

4

Configura DNS para el dominio

Apunta tu dominio a la IP pública o configura Azure DNS según tus necesidades.

5

Para alto tráfico: añade balanceador

Implementa un Application Gateway o Load Balancer para distribuir el tráfico entre múltiples instancias.
